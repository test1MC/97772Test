System Requirements

Hardware and Software Requirements

InterMapper runs on Windows, MacOS X, and various Linux distributions (see below.) Solaris versions are no longer supported.

InterMapper RemoteAccess runs on MacOS X, Windows, and Unix (including Linux) computers. Java or newer required. In general, any computer capable of running MacOS X, or a 1 GHz Pentium or faster Windows or Unix computer with 512 Mb of RAM will serve quite nicely to run InterMapper RemoteAccess.

InterMapper Flows requires a fairly powerful computer. Both processor power and the amount of RAM are important considerations. Although you may perform initial testing using a VM (virtual machine) environment, we do not recommend that you use a VM in production because the performance is frequently not good enough.

InterMapper Flows benefits from having a large amount of RAM for its database cache. If you want to allocate more than 1.5Gb of RAM, use a 64-bit Windows, Macintosh, Linux, or Unix system.

Operating System Requirements

InterMapper for MacOS X requires MacOS X or MacOS X Server, . Any computer that can run this version of MacOS X will easily handle large maps.

InterMapper for Windows 2003/Windows Server 2008/2012/Windows 7/8/8.1 A Windows computer with at least a 1GHz Pentium and 512Mb of RAM will work well for at least 500 devices. Larger installations will benefit from a faster processor or more RAM. InterMapper support for IPv6 requires Windows Server 2008, or newer.
Notes:

  • XP - InterMapper is no longer supported.
  • Vista - Use of InterMapper with Vista is strongly discouraged and not supported.

InterMapper for Linux has been tested with the following Linux distributions:

  • Debian
  • SuSE Enterprise Linux
  • OpenSuSE
  • Red Hat Enterprise Linux ;
  • Ubuntu

InterMapper requires an x86 processor with 512Mb of RAM.

InterMapper for Solaris is no longer supported.

Java - all versions of InterMapper require Java Runtime Environment (JRE) version or . In Windows and Mac versions, JRE is installed. If you choose not to use the JRE, is required.

InterMapper Connection Policy

  • InterMapper allows an unlimited number of clients to connect. However, InterMapper does check the serial numbers of paid, full copies of InterMapper RemoteAccess, and will not allow two copies with the same serial number to connect.
