Network Device Probes

InterMapper has a number of probes that use SNMP to retrieve information from a device being tested. These include:

  • Apple Probes
    • Apple AirPort Extreme - Tests an Apple AirPort Extreme Base Station
    • Apple AirPort (Graphite) - Tests an Apple AirPort Base Station (version 1, Graphite)
  • Cisco
    • IP SLA Jitter -Uses SNMP to collect the information from the RTTMON MIB in a device configured as an IP SLA agent, allowing you to alarm jitter, latency, and packet loss, and to chart these values.
    • Old Cisco CPU MIB - Checks free memory and CPU utilization of a Cisco router
    • Process and Memory Pool- monitors the CPU and Memory utilization of a Cisco router.
    • Cisco Aironet Probe - Tests Cisco Aironet access points
  • Juniper Netscreen VPN (Pg 1) - monitors the status of VPN tunnels in a Netscreen Firewall.
  • Karlnet Wireless - Tests a Karlnet Wireless Base Station
  • UPS Probes - Probes for uninterruptible power supplies.

If a device does not respond to SNMP

InterMapper uses SNMP (Simple Network Management Protocol) to learn about attached networks and to create the network clouds you see on the map. When a device doesn't answer SNMP, InterMapper automatically probes it with the standard echo probes, so you can still monitor the device.

Here are some reasons your routers might not answer:

  • The device might not respond to SNMP queries with a community string of "public".
  • There might not be an SNMP agent running in the device.
  • The device might not be reachable.
  • The machine running InterMapper might not be entered in the device's access list.

A community string is akin to a login-name for a router. SNMP uses community strings to specify what variables a client can read or modify. A network manager often sets a community string of "public" to permit read-only access to SNMP data. If your site uses a different community string for read-only access, you must specify this when adding a device, or the device will not respond.

Apple Probes

Apple AirPort (Extreme)

InterMapper queries an Apple AirPort Extreme Base Station. In addition to monitoring the traffic statistics (using the SNMP probe above), InterMapper displays the number of wireless clients associated with the base station along with their MAC address/user name, and the signal strength in dBm.

For the AirPort (Extreme) "UFO" version, use its password for the probe's community string. For AirPort (Extreme) "Square", use the community string defined for the unit.

Apple AirPort (Graphite)

InterMapper queries an Apple Graphite AirPort Base Station (version 1). In addition to monitoring the traffic statistics (using the SNMP probe above), InterMapper displays the number of wireless clients associated with the base station along with their MAC address/user name, and the signal strength and noise figures (0-100%).

The parameter to the SNMP-AirPort probe is the password to the AirPort Base Station.

Note: The Snow AirPort Base Station (version 2) does not provide the SNMP information about associated clients. Help/Systems is working with Apple to find a way to address this.

Cisco Probes

Cisco CPU Probe

InterMapper queries the CPU utilization and free memory of a Cisco router and places the device in warning or alarm based on user-settable thresholds.

 

Cisco IP SLA Jitter Probe

This probe uses SNMP to collect the information from the RTTMON MIB in the agent, allowing you to alarm jitter, latency, and packet loss, and to chart these values. For more information, see Cisco IP SLA Probe.

Cisco Process and Memory Pool Probe

This probe monitors the CPU and Memory utilization of a Cisco router using variables from CISCO-MEMORY-POOL-MIB and CISCO-PROCESS-MIB. Two thresholds (Alarm and Warning) are available for CPU utilization , and two are available for low memory.

Cisco Aironet Probe 

InterMapper queries a Cisco Aironet wireless access point. In addition to monitoring the traffic statistics (using the SNMP probe above), InterMapper displays the number of wireless clients associated with the base station, each user's MAC address and user name, and the unit's signal strength and noise figures (0-100%).

The parameter to the SNMP-Aironet probe is the SNMP read-only community string.

Juniper Netscreen VPN

This probe monitors the status of VPN Tunnels in a Netscreen Firewall. It uses the nsVpnMonTable to monitor the Netscreen's active tunnels, which are treated and mapped as separate interfaces.

Some statistics may be available only if the monitoring status is on for the tunnel, as reported by nsVpnMonMonState.

Karlnet Wireless

This probe monitors the Karlnet Wireless Base Station's custom MIB using SNMPv1. It reports the number of clients using the base stations, and the signal strength of each. It uses SNMP set-requests to tell the Karlnet Base Station to discover its clients periodically, and to test each client's signal strength. It uses SNMPv1 and requires a read/write community string.

UPS Probes

A number of probes are available to support uninterruptible power supplies from APC, BestPower, Exide, Liebert, Standard, TrippLite, and Victron.

 

Network Devices/UPS/APC UPS

For APC UPS devices.

A. Probed MIB(s)

This probe works best with devices which have implemented the listed MIB(s).

(1 of 1) APC UPS MIB [... enterprises.apc.products.hardware.ups / ... 1.3.6.1.4.1.318.1.1.1]

B. Displayed Values

UPS: model, firmware, status, (battery: capacity, time remaining, temperature, replacement status), (output: load percent, volts, amps, frequency), (input: volts, voltage range over last minute, frequency, last input failure).

C. Alarms

(1 of 3) If unit goes onto battery or goes off-line.

(2 of 3) If battery needs replacement.

(3 of 3) If the battery temperature exceeds user-specified thresholds (see "Parameters" below).

D. Warnings

(1 of 2) If unit goes onto "Smart Trim" or "Smart Boost"

(2 of 2) If the battery temperature exceeds user-specified thresholds (see "Parameters" below).

E. Parameters

(1 of 5) Units of Temperature (C / F): Determines how the following thresholds are interpreted.

(2 of 5) Alarm Threshold - Low Temp: Threshold for alarm state (see above).

(3 of 5) Warning Threshold - Low Temp: Threshold for alarm state (see above).

(4 of 5) Warning Threshold - High Temp: Threshold for warning state (see above).

(5 of 5) Alarm Threshold - High Temp: Threshold for warning state (see above).

Bugs or Requests? Please contact us.

Filename: com.dartware.ups.apc.txt
Version: 3.4

Network Devices/UPS/Victron UPS

Victron UPS

This probe monitors some needfull USV parameters.

UPS Battery Status - Alarm is the ALARM threshold for Battery Status.

If the value is 2 USV working normal.

If the value is 1 USV is on bypass, the device will enter the Alarm state.

UPS Battery Remaining - Warning is the WARNING threshold for the estimated battery time remaining. If the Battery Remaining is less than this threshold, the device will enter the Warning state.

UPS Battery low Voltage - Warning is the WARNING threshold for the min. battery voltage. If the Battery voltage is less than this threshold, the device will enter the Warning state.

Low Input Voltage line 1 - Alarm is the ALARM threshold for min input voltage on phase 1. If the input voltage drops below this threshold, the device will enter the Alarm state.

Low Input Voltage line 2 - Alarm is the ALARM threshold for min input voltage on phase 2. If the input voltage drops below this threshold, the device will enter the Alarm state.

Low Input Voltage line 3 - Alarm is the ALARM threshold for min input voltage on phase 3. If the input voltage drops below this threshold, the device will enter the Alarm state.
