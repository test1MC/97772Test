Server Probes - Proprietary

The following is a list of probes that test vendor-specific devices, or devices that require proprietary protocols.

  • 4D Server - Tests a 4D Server
  • Apple Servers
    • AppleShare IP - Tests an AppleShare IP server
    • OS X Server
      • AFP - Tests an OS X AFP server
      • FTP - Tests an OS X FTP server
      • Info - Tests an OS X Info server
      • NAT - Tests an OS X NAT server
      • Print - Tests an OS X Print server
      • QTSS - Tests an OS X QTSS server
      • Web - Tests an OS X Web server
    • RTMP - Tests an AppleTalk router
    • Xserve
      • Xserve g4 - Tests an Xserve G4 server
      • Xserve g5 - Tests an Xserve G5 server
      • Xserve Tiger - Tests an Xserve Tiger server
      • Xserve Raid - Tests an Xserve RAID
  • Barracuda Probes
    • Barracuda HTTP - Tests a Barracuda Spam Firewall using HTTP
    • Barracuda HTTPS - Tests a Barracuda Spam Firewall using HTTPS
  • Big Brother - Listens for reports from Big Brother clients on remote servers.
  • BlitzWatch - Tests a BlitzMail server
  • Citrix Server - Tests a Citrix server.
  • DND Protocol- Tests a DND server
  • FileMaker Pro - Tests a FileMaker Pro server
  • FirstClass Server - Tests a Centrinity FirstClass server
  • KeyServer® - Tests a KeyServer from Sassafras Software
  • Lotus Notes - Tests a Lotus Notes server
  • MeetingMaker- Tests a MeetingMaker server
  • Windows NT Services - Tests a Windows NT service

 

4D Server

InterMapper connects to a 4D Server on port 19813, and examines the response for the specified database name.

Apple Probes

AppleShare IP

InterMapper sends an AppleShare "Get Server Info" request through a TCP connection to the specified port (default is port 548) on the device being tested. The device is shown as DOWN if the connection cannot be completed or if an error response is returned.

Note: AppleShareIP versions prior to 6.3 may have a bug where the AppleShareIP server hangs if it receives an InterMapper Get Server Info probe during startup. Version 6.3 and newer do not seem to have this problem.

A request for status information is made via an HTTPS post to the Server Admin port. The server responds with XML data that is then parsed by the probe.

RTMP

InterMapper sends an AppleTalk RTMP RDR Request query of type 3, and waits for a RTMP response.

XServe Servers

InterMapper can monitor an Apple Xserve via the same facilities used by Apple's Server Monitor application. There are significant hardware differences between the various servers that require separate probes for G4 Xserve and G5 Xserve computers. The G4 Xserve also requires that the version of Mac OS X running on the machine be specified.

The probe sends a POST request via HTTPS to port 311. The response to this request is an Apple plist in XML form. The XML is then parsed by the probe to retrieve the information. Probes require an administrators name and password in order to access the information.

XServe G4

This probe uses TCP to query an Xserve G4 server using the Server Monitor port and protocol. It uses HTTPS post to make a status request to the Server Monitor port. The server responds with XML data that is parsed by the probe.

Xserve G5

This probe uses TCP to query an Xserve G5 server using the Server Monitor port and protocol. It uses HTTPS post to make a status request to the Server Monitor port. The server responds with XML data that is parsed by the probe.

Xserve Tiger

This probe queries and Xserve running Mac OS X 10.4 using the Server Monitor port and protocol. It uses HTTPS post to make a status request to the Server Monitor port. The server responds with XML data that is parsed by the probe.

Xserve Raid

This TCP probe queries an Xserve RAID for various details using the RAID Admin port and protocol. It uses the same facilities used by Apple's RAID Admin application. The probe requires the same monitoring password used by RAID Admin to access the status information.

The probe sends a POST request via HTTP to port 80. The response to this request is an Apple plist in XML form. The XML is then parsed by the probe to retrieve the information.

Apple has set pre-configured thresholds for several different states. These thresholds are not editable, but the RAID Admin protocol will indicate when these thresholds are exceeded and an error message will be displayed by the probe.

Barracuda Probes

Barracuda HTTP

This probe uses TCP to query a Barracuda Spam Firewall to get various performance statistics. An HTTP GET request retrieves the BASIC->Status page of the Administrator's interface.

Barracuda HTTPS

This probe uses TCP to query a Barracuda Spam Firewall to get various performance statistics. An HTTPS GET request retrieves the BASIC->Status page of the Administrator's interface.

Big Brother Probe

InterMapper can act as a Big Brother server, and will listen for reports from Big Brother clients installed on remote computers. For more information, see Big Brother Probes.

BlitzWatch

InterMapper sends BlitzMail® status queries to a BlitzMail server, and checks for appropriate responses. BlitzMail is a high performance, cross platform client-server e-mail system designed at Dartmouth College. More information is available at http://www.dartmouth.edu/netsoftware/.

Citrix Server

This probe connects to a Citrix server that defaults to listening on port 1494. It checks the received response for the presence of "ICA", indicating that the Citrix server is running.

This probe goes into alarm condition if:
- it receives an unexpected disconnect
- it connects, but doesn't receive a response after 30 seconds
- the response doesn't contain the string "ICA"

DND Protocol

InterMapper connects to a DND (Dartmouth Name Directory) server on port 902 and issues a request for the individual named in the parameter.

FileMaker Pro

InterMapper connects to a FileMaker PRO server on the default port 5003, and returns OK if the connection is successful.

FirstClass Server

InterMapper connects to a port (default is 510) on a FirstClass e-mail server. It sends two CR's, and examines the resulting response for the presence of the specified banner string.

KeyServer®

InterMapper sends a KeyServer® Status request to the device, and waits for an appropriate response. The format of this packet is available from Sassafras Software, http://www.sassafras.com

Lotus Notes

Lotus Notes uses Port 1352 for its Remote Procedure Call and Notes Replication. This probe establishes a connection to the specified port on a Lotus Notes server. If the connection is successful, the device's status is set to OK. Otherwise its status is set to DOWN.

MeetingMaker

A MeetingMaker server listens on port 649 (by default). InterMapper attempts to connect, and returns an UP status if it is successful.

NT Services Probe

If your InterMapper server is running on an NT-class computer (XP and beyond), InterMapper logs onto a specified machine and checks to see if a specified service is running. For detailed information, see Monitoring Services with the NT Probe
