Miscellaneous Probes

Use these probes for a variety of uses. You can find the Demo, Non-Polling, and TCP Check probes. You can also find the Legacy probes (included to support older maps) and the template for developing Nagios and Command-line probes.

  • Demo probe - Use this probe to create demonstration maps, which simulate a network and its activity.
  • Legacy Probes - These probes have been deprecated. They are included to support older maps.
  • Nagios Probes - Use this probe to specify a Nagios plugin.
  • Non-polling probe - Choose this probe so that the selected device is not probed.
  • Prototype SNMP Probe- Use this probe as the basis for creating custom SNMP probes.
  • TCP Check Probe - checks the number of TCP connections and sends a notification if it is exceeded.

Demo Probe

The Demo Probe is designed solely for making test maps. InterMapper creates a device with the specified number of links and the specified percentage of interconnections. InterMapper then generates random data for each of its interfaces, and randomly sets the device status to UP or DOWN.

Note: Some InterMapper licenses specify a number of devices you can monitor. Demo probes do not reduce the number of devices available for monitoring.

Legacy Probes

The following probes have been deprecated, but are kept available for older maps that may still be using them.

  • SNMP - Basic OID (v2c) - Sends SNMP queries for a specified OID
  • Cisco v2c Probe - same as the Cisco Probe, but uses 64-bit counters as required
  • SNMP v2c - Probes MIB-II variables, with 64-bit counters

SNMP - Basic OID (v2c) Probe

InterMapper queries the specified OID (Object ID) using a SNMPv2c Get-Next-Request. It can interpret the full range of variable types specified by SNMP v2c. The status window displays the value of the requested variable.

Cisco (v2c) Probe

This probe is identical to the Cisco probe, except that it uses SNMP v2c queries to use the 64-bit counters.

SNMPv2c Probe

This probe uses SNMPv2C queries to get statistics from the device. This is primarily intended to monitor the 64-bit counters present on very high speed links.

Nagios Plugins

Use the Nagios probe type to select plugins from the Nagios monitoring system. InterMapper can use these plugins to test devices. For more details, see Nagios Plugins in the Developer Guide.

Non-Polling Probe

When the Non-Polling probe is selected, InterMapper does not poll the device. Its icon becomes gray, and InterMapper's map no longer reflects the device's state.

A device that used a non-polling probe is not counted against InterMapper's device count.

Prototype SNMP Probe

This probe demonstrates an InterMapper Custom SNMP probe, and many of the features that are described in the Custom Probes section of the Developer Guide. 

If you have questions about this probe, please contact us.

This probe probably isn't really useful for production work. However, it provides an example of various techniques available in custom SNMP probes. It shows how to retrieve SNMP values from a device by specifying their OIDs and how to display those values in the device's Status Window. The probe also has thresholds for setting the device into alarm or warning.

TCP Check Probe

This probe generates an alarm if the count of TCP connections is greater than the "Allowed TCP Connections" parameter. It can be used to detect people telnetting into a box that shouldn't have connections. An example is a router that might be attacked from outside your network.

InterMapper queries the MIB-II TCP table to retrieve tcpCurrEstab (1.3.6.1.2.1.6.9.0), which measures the number of TCP connections in place. If that count exceeds the parameter, the device is placed in alarm.
